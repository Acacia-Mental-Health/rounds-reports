1. Purpose

This SOP was created to describe in detail the steps involved in creating rounds reports for patients at Acacia Clinic.

2. Scope

This document describes the creation of the rounds reports with the program created originally by Emily Jusef and heavily updated by Pierce Wang during the summer of 2022. The script creates an HTML form round report for each person, but should be double checked by its user for correctness as user error in data entry can often be an issue.

3. Goals

Standardize the process of creating rounds reports and inform those relevant persons on how to set up their environment and use the program.

4. Definition

Terminal
A Unix-based bash/zsh shell which allows one to run commands on a command line
bash/zsh
Programs that run within a shell which can interpret and call other programs (such as cd listed below)
cd
A command/program on unix-based systems. This command stands for change directory
R
A program/programming language that is often used for bulk data analysis.
R shell
The interface which is called when you run the R command in a UNIX terminal/Windows shell.
RStudio
An application which is used for programming in R and provides useful data.
Rscript
A program (called on the command line) that runs files with the extension .R line by line by feeding it into the R shell

5. Responsibilities

  • Program initially Emily Jusef and then heavily updated/improved by Pierce Wang, responsibilities for code are thus assigned accordingly.
  • Additionally, human error sometimes causes discrepancies in data.
  • Typically one person in the clinic is assigned to create the rounds reports for weekly meetings between TMS technicians and providers, thus this document describes the process that they should go through in creating those reports.

6. Procedure

The following prescribes the order in which you should follow these instructions:

  • If you are setting your system up for the first time, then follow the directions step by step through the Initial Setup instructions first, and then step through Running the program
  • If you have already set up the system on your computer, then follow the directions in Editing the Information Input File, then go ahead to Running the program
  • If you’re running into problems with authentication, then follow the instructions in Google Re-authentication

        2. Navigate to your Applications folder
        3. Navigate to the Utilities folder
        4. Double click on the Terminal application
    2. Check that R is installed correctly Type R --version into terminal and press enter
    3. If it returns a version number, then you’re all set up!

6.1.3. R Google Account Integration Setup

  1. Open terminal (see Terminal/Command Line Setup for help)
  2. Open R shell Type R then pressing enter
  3. Install googlesheets4 package (type and press enter at the end of each line to send it to the shell)
    1. install.packages("googlesheets4") Type this line in, press return, and then you will choose a mirror (by typing the corresponding number and pressing enter). Pick any one of these locations. Preferably, pick one that is closest to you, but the package is small enough that it doesn’t matter that much. So, pick whichever one you feel like! Follow the rest of the prompts to finish installing the package, then continue to the next step.
    2. library(googlesheets4) This command will not provide any output. It simply loads the package you just downloaded and makes the appropriate functions available to you.
    3. gs4_auth() This function will bring up a few prompts. Say yes to all of them by pressing 1 and then return for each.
    4. Now that everything is set up, type q() and press n to leave the R shell
  4. Once you run the last command of the previous step, you will be given a couple options. If the email you wish to connect to is listed there, then type the corresponding number. If not, then type 0 and press enter to authenticate a new account

6.1.4. Directory setup

  1. Download the github repository (source code) from TMS Rounds Reports Generation Download Link
  2. Unzip the zip file
  3. Move the folder to an accessible place (you decide where to put it such that it would be most organized) on your computer

  • The ProgressReportSetup.R file is the one which you will edit with appropriate patient codes and time constraints.
  • The ProgressReportScript.R file is the one which you will edit with appropriate patient codes and time constraints.
  • The ProgressReportsGenerator.R file is the one you will run once everything is set up.

6.2. Recurring Setup and Debugging

6.2.1. Editing the Information Input File

This step is to be repeated every time you generate notes according to which patients and what time period you want to generate notes for.

  1. In any text editor (including RStudio), open up the ProgressReportSetup.R file and edit the following two variables:
    list_to_generate

    A list of patient codes defining which patients to create reports for. The script searches the demographics sheet on the Google drive, so the code needs to be correctly capitalized (as it is used at the top of the report) but it only needs to exist in the first column of the patient demographics column. 

    list_to_generate <- list(
  list("AaBb11"), # Patient code can be any code which, when all letters are
                  # lowercased, matches a patient code in the patient
                  # demographics sheet on Google Drive
  list("AaBb12", "2021-05-22"), # Include only those entries after this date
  list("AaBb13")
)
    rounds
    A boolean (TRUE/FALSE) variable defining whether the patient’s demographics, session number, and planned course will show at the top of the report.

6.2.2. Google Re-authentication

This step is only necessary if the authentication is not working.

  1. Open terminal (see Terminal/Command Line Setup for help)
  2. Open R shell Type R then pressing enter

  3. Authenticate for the appropriate google account 

    library(googlesheets4)
gs4_auth()
  4. Once you run the last command of the previous step, you will be given a couple options. If the email you wish to connect to is listed there, then type the corresponding number. If not, then type 0 and press enter to authenticate a new account.

6.3. Running the program

Once all setup has been completed, follow these instructions to run the program.

  1. Navigate/Go to the folder where you have stored the rounds-reports directory Note: You should not be in the directory at this point. Rather, you should see the folder rounds-reports. This is important for the next step
  2. Open terminal
  3. Type cd + space, then from Finder drag the folder into Terminal then type enter
  4. Now, type Rscript src/ProgressReportScript.R + enter
    • NOTE: If the script is not working, and if the error log shows it has anything to do with authentication with Google, then refer to the steps for Google Authentication: R Google Account Integration Setup & Google Re-authentication
